## 🎯 What is LexAI?

LexAI is a **fully functional, no-mock-data, production-quality legal operating system** built for advocates and lawyers. Every button triggers real backend APIs. Every feature produces real computed output. No hardcoded data. No fake integrations.

### ✨ Key Highlights

✅ **Real Case Management** - Create, store, analyze cases with AI
✅ **AI-Powered Insights** - LLaMA3/Mixtral + OpenAI fallback
✅ **Document Intelligence** - Upload, extract, summarize, verify documents
✅ **Legal Drafting** - Generate notices, petitions, replies with tone control
✅ **Similarity Search** - Find similar cases using RAG + embeddings
✅ **Client Portal** - Manage clients and their cases
✅ **Analytics Dashboard** - Real metrics from PostgreSQL
✅ **Blockchain Verification** - SHA-256 document hashing
✅ **Multi-step Workflows** - Case creation, document upload, analysis in one flow

## 🚀 Quick Start

### Option 1: Docker (Recommended)

```bash
# Clone and navigate
git clone <repo>
cd LEXIS

# Start with Docker Compose
docker-compose up -d

# Access
# Frontend: http://localhost:3000
# Backend: http://localhost:8000
```

### Option 2: Manual Setup

**Prerequisites**: Python 3.10+, Node.js 18+, PostgreSQL 12+

```bash
# Backend
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env  # Configure your keys
uvicorn main:app --reload

# Frontend (new terminal)
cd frontend
npm install
npm run dev
```

See [SETUP.md](./SETUP.md) for detailed configuration.

## 🎮 Features in Action

### Create a Case
1. Click **"New Case"** in sidebar
2. **Step 1**: Enter title, description
3. **Step 2**: Upload documents (auto-extracts text)
4. **Step 3**: Review & submit
5. ✅ AI analyzes case, finds similar cases, generates summary

### Ask AI Assistant
1. Navigate to **AI Assistant**
2. Type question (e.g., "What are strongest arguments?")
3. AI retrieves similar cases via RAG
4. LLM generates answer with context
5. View strategies, risks, and references

### Generate Legal Document
1. Go to **Legal Drafting**
2. Select document type (Notice/Petition/Reply)
3. Fill dynamic form
4. Choose tone (Formal/Aggressive/etc.)
5. AI generates document → Download or send

### Manage Documents
1. Upload to **Document AI**
2. Auto-summarize with AI
3. View extracted clauses and risks
4. Verify document hash
5. See in case context

## 📈 Performance Considerations

- **RAG Index**: FAISS in-memory (5000+ cases ~100MB)

- **Document Processing**: ~2-10s depending on size
- **Database**: Indexed queries, async I/O
- **Frontend**: Client-side rendering, optimized bundle

### Production Optimizations
- Persist FAISS to disk
- Use Redis for caching
- Connection pooling
- CDN for frontend
- Load balancing
- Database replication
